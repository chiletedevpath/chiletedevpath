## 1. Objetivo

Establecer los criterios editoriales para publicar contenido en Chilete DevPath con claridad, honestidad técnica, respeto por la autoría y utilidad real para la comunidad.

## 2. Alcance

Esta política aplica a:

- `aprendizaje`
- `academia`
- `colaboraciones`
- `portafolio`
- repositorios académicos, personales y formativos publicados bajo Chilete DevPath
- guías, ejercicios, laboratorios, soluciones, fichas, documentación técnica y materiales de apoyo

## 3. Políticas relacionadas

- [Política de Uso Responsable de IA](POLITICA_USO_RESPONSABLE_IA.md)

## 4. Definiciones

| Término | Definición |
|---|---|
| Contenido propio | Material creado directamente para Chilete DevPath. |
| Contenido inspirado | Material basado en una idea externa, pero adaptado con estructura, ejemplos o ejercicios propios. |
| Fuente externa | Documentación, curso, video, libro, repositorio, artículo o recurso creado por terceros. |
| Material académico privado | Evaluaciones, rúbricas, enunciados internos o recursos cerrados de una institución, docente o curso. |
| Colaboración | Proyecto o contenido donde participa Chilete DevPath, pero cuya autoría o repositorio principal pertenece a otra persona o equipo. |

## 5. Responsabilidades

| Rol | Responsabilidad |
|---|---|
| Fundador de Chilete DevPath | Definir, revisar y actualizar esta política. |
| Autor del contenido | Verificar autoría, citar fuentes y validar que el material pueda explicarse con palabras propias. |
| Comunidad lectora | Usar el contenido como apoyo formativo y respetar las fuentes citadas. |
| Colaboradores | Reconocer autorías externas y declarar claramente el alcance de su participación. |

## 6. Lineamientos editoriales

Chilete DevPath debe publicar contenido que:

- Explique qué se está aprendiendo o construyendo.
- Diferencie contenido propio, inspirado y basado en fuentes externas.
- Cite fuentes cuando se usen ideas, ejercicios, documentación, videos, libros, cursos o repositorios externos.
- No publique enunciados privados completos, rúbricas cerradas o material interno de evaluación cuando no corresponda compartirlo públicamente.
- No presente como propio el trabajo de otras personas.
- Reconozca colaboraciones, repositorios externos y equipos originales.
- Sea claro para estudiantes que están aprendiendo desde fundamentos hasta temas avanzados.

## 7. Presentación del contenido

| Tipo de contenido | Presentación requerida |
|---|---|
| Creado para Chilete DevPath | Indicar que el ejercicio, guía o laboratorio fue creado para esta ruta. |
| Inspirado en una fuente | Mencionar la fuente y aclarar que el contenido fue adaptado. |
| Basado en documentación oficial | Enlazar la documentación consultada. |
| Basado en una clase o curso | Registrar institución, curso o contexto cuando se pueda compartir públicamente. |
| Relacionado con evaluación o material privado | No publicar el enunciado completo; crear una versión propia o documentar aprendizajes generales. |
| Proyecto colaborativo | Reconocer el repositorio, autoría o equipo original cuando corresponda. |

## 8. Fuentes y referencias

Cuando un material use una fuente externa, debe incluir una sección llamada `Fuentes` o `Referencias`.

Formato base:

```md
## Fuentes

- Autor o institución: título del recurso. URL o referencia consultada.
```

Formato para contenido adaptado:

```md
## Fuentes

- Inspirado en: nombre del recurso o curso. Adaptado con ejercicios propios para Chilete DevPath.
```

## 9. Uso de IA

Cuando se use IA para apoyar una explicación, crear ejercicios o revisar redacción, el material debe aclarar que fue revisado y validado antes de publicarse. El criterio completo se define en la [Política de Uso Responsable de IA](POLITICA_USO_RESPONSABLE_IA.md).

Formato base:

```md
## Nota de autoría

Contenido creado para Chilete DevPath con apoyo de IA para estructuración o revisión. El criterio, selección y validación final corresponden al autor de Chilete DevPath.
```

## 10. Restricciones

No se debe publicar:

- Contenido de terceros sin referencia.
- Soluciones copiadas sin comprensión.
- Evaluaciones privadas completas.
- Información sensible, credenciales o datos personales.
- Material que simule autoría propia cuando pertenece a otra persona o herramienta.

## 11. Excepciones

Toda excepción a esta política debe justificarse en el README, ficha o documento correspondiente. Si existe duda sobre si un contenido puede publicarse, se debe priorizar una versión propia, resumida o enfocada en aprendizajes generales.

## 12. Revisión y actualización

Esta política debe revisarse cada seis meses o cuando cambie la forma de publicar contenido, colaborar, usar IA o documentar fuentes dentro de Chilete DevPath.
